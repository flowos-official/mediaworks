# MediaWorks 商品発掘システム 利用ガイド

매일 아침의 루틴부터 주간 리뷰까지, 전체 시스템 사용 흐름.

---

## 🌅 매일 아침 (평일 09:00 이후, 약 10-20분)

### Step 1. 발굴 결과 확인

**URL**: `/ja/analytics/discovery/home` (기본 홈쇼핑)

매일 자동으로 생성된 **30개 신상품 후보**가 표시됩니다:
- 09:00 JST → ホームショッピング 발굴 자동 실행
- 09:30 JST → ライブコマース 발굴 자동 실행

상단에 다음 정보:
- 🟢 **완료** 배지 + 실행 시간
- `30/30 件` (목표 대비 실제 수)
- `ソーシング済み: N` (과거 누적)

### Step 2. 30개 카드 리뷰 (1-2분/카드)

각 카드에 다음이 포함됨:
- 🖼️ **썸네일** (Rakuten 이미지)
- 🏪 **楽天 배지** + **제품명** + **tv_fit_score** (100점 만점)
- 💰 가격, ⭐ 리뷰 평점+개수, 🛒 판매자
- 🏷️ **TV実績** / **探索** 배지
- ⚠️ broadcast tag (경쟁사 방송 이력 있으면)
- 💡 **TV適合性** 노랑 박스 (왜 홈쇼핑에 적합한지 AI 설명)

### Step 3. 빠른 피드백 (한 카드당 2-5초)

**4개 버튼 중 하나 클릭**:
- ✅ **ソーシング済み** (녹색) — "이 상품 소싱할 것" → 향후 발굴에서 **영구 제외**
- ⭐ **関心あり** (주황) — "검토 가치 있음" → 학습에 양성 신호
- ❌ **却下** (빨강) + 모달 이유 선택 → 해당 URL/브랜드 **rejected_seeds** 등록
- 🗑️ **既にあり** (회색) — "이미 가지고 있는 상품" → 영구 제외

**토글**: 같은 버튼 다시 클릭 → 해제.

---

## 🔍 관심 상품 심화 분석 (필요 시)

### Step 4. "深掘り" 클릭

관심 있는 상품 카드의 **🔍 深掘り** 버튼 클릭:
- 비동기 실행 (15-45초)
- Gemini 에이전트가 자동으로 도구 호출:
  - Rakuten 판매자 페이지 크롤링
  - 제조사 공식 사이트 확인
  - 도매가/마진 추정 계산
  - TV 방송 30초 스크립트 생성
- 완료되면 버튼이 **"詳細を見る"** 로 변경

### Step 5. C 패키지 Drawer 확인

버튼 클릭 시 카드 하단 drawer 확장:
- 🏭 **제조사 정보** + 공식사이트 + 연락처 힌트
- 💰 **도매가 추정** (¥X,XXX / 마진 Y%, 신뢰도 표시)
- 📋 **MOQ 힌트** (있는 경우)
- 🎬 **TV 방송 스크립트 초안** (복사 가능)
- 🔥 **SNS 트렌드** 신호 (high/medium/low/none)

## 📊 전략 생성 (소싱 결정 시)

### Step 6. 拡大戦略 / ライブ戦略 생성

카드의 **📈 拡大戦略を作成** / **📡 ライブ戦略を作成** 버튼 클릭:

**Case A: 深掘り 안 했으면** → 팝업 표시
- "深掘り情報なし — 자동 실행?"
- 2개 버튼:
  - **スキップ(簡易戦略)**: B 패키지만으로 전략 생성
  - **深掘りして戦略作成**: enrichment 먼저 실행 후 전략 생성

**Case B: 深掘り 완료됨** → 바로 전략 페이지 이동

### Step 7. 전략 페이지에서 생성 시작

`/ja/analytics/strategy/expansion?seedId=<UUID>` 자동 이동:
- **userGoal** 텍스트박스에 자동 pre-fill (제품명 + 참조 URL)
- **price帯** 도 pre-fill
- **拡大戦略を分析** 클릭 → Gemini 6 skills 순차 실행 (~60-120초)

결과:
- **신상품 실제 데이터 기반 맞춤 전략** (가격/마진/채널/리스크 분석)
- AI 분석 근거: 기존 `product_summaries` (TV 판매 실적) + seed 제품 데이터 + SNS 트렌드
- 전략 저장 후 나중에 재조회 가능 (`/analytics/strategy/expansion/[id]`)

- **필터**: すべて / ホーム / ライブ + 월 이동 ← →

## 📈 주간 인사이트 (월요일마다)

### Step 9. `/ja/analytics/discovery/insights`

두 개의 서브 탭:

**A. 選別 탭** — 과거 피드백 리스트
- 상태별 필터 (ソーシング済み / 関心あり / 却下 / 既にあり)
- Context 필터 (ホーム / ライブ)
- 기간 필터 (7日 / 30日 / 90日)
- 페이지네이션 (20개/페이지)

**B. 統計 탭** — 주간 분석
- **KPI 4 카드**:
  - 今週のソーシング 수
  - 今週の却下 수
  - 探索比率 현재값
  - 총 피드백 샘플 수
- **週間インサイト (Gemini AI)**:
  - 🎯 成功パターン: "이번주 소싱한 상품의 공통 특성"
  - 🧭 探索成果: "탐색 카테고리에서의 성과"
  - 💡 来週の提案: "다음주 전략 제안 (구체적 카테고리/비율)"
- **차트 4종**:
  - 카테고리별 소싱률 (가로 막대, 홈/라이브 구분)
  - 일별 피드백 활동 (스택 막대, 30일)
  - 탐색 비율 추세 (꺾은선, 12주)
  - 거절 이유 분포 (도넛)

## 💡 효율적 사용 팁

### 하루 루틴 권장
```
09:00 → 사이트 열기
09:00~09:20 → 30개 홈 카드 리뷰 + 피드백
09:20~09:40 → 30개 라이브 카드 리뷰 + 피드백
(09:40~) → 관심 3-5개 深掘り 동시 실행 (비동기)
10:00~ → 그 중 1-2개 戦略を作成 → 상세 분석
```

### 피드백 뉘앙스
- **ソーシング**: 실제로 소싱할 결정 (영구 제외 + 학습 양성)
- **関心あり**: "나중에 다시 보자" (학습 양성, 미제외)
- **却下**: 정말 안 맞음 (제외 + 학습 음성)
- **既にあり**: 실수 아니고 중복 (제외만, 학습은 영향 없음)

### 주간 리뷰 (월요일 아침)
- 인사이트 탭 통계 확인
- 来週の提案 을 참고로 관심 카테고리 조정
- 必要시 탐색 비율 수동으로 느낌 확인 (자동 학습도 진행 중)

## 🆘 문제 해결

### 이미지가 안 보임
→ 이전에 발굴된 데이터 (Phase 3.5 이전). 다음 cron 이후 자동 해결.

### 발굴 결과가 이상함
→ 인사이트 탭에서 "却下理由" 확인 → 특정 이유가 많으면 학습이 자동 조정 중.

### 전략 생성이 오래 걸림
→ 정상 (60-180초). 브라우저 닫아도 전략은 DB에 저장됨, `/analytics/strategy/expansion/[id]` URL로 돌아갈 수 있음.

### 깊이 파기 실패
→ 부분 결과라도 저장됨. "詳細を見る" 클릭 시 "部分結果" 배지 표시.
